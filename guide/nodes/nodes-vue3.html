# 过滤器 filters

过滤器的主要用于过滤我们的数据，让数据按照我们规定的格式来做展示

  1. 过滤器要先定义后使用

  2. 过滤器中的方法必须有return，如果没有，页面什么都不会渲染

  3. return返回的数据就是过滤后的数据，经过返回后，就会渲染到页面上

  4. 过滤方法的形参用于接收要过滤的数据，过滤方法在调用时不需要加()

# 自定义指令 directives

自定义指令名在定义的时候，不需要使用 v-

bind函数的触发时机：当这个自定义指令被使用时就会自动触发 update函数的触发时机：当自定义指令的指令值发生改变的时候会自动触发该函数

//1.声明语法格式
new Vue({
	directives:{
		'自定义指令名':{
			bind(el,binding){
				//el参数：绑定该指令的标签
				//binding参数：是一个对象
			},
			update(el,binding){
				//el参数：绑定该指令的标签
				//binding参数：除value值外，其它参数与bind的binding一样
			}
		}
	}
})

//2.使用自定义指令

# 深度监听

深度监听：deep:true

我们监听的数据不能是对象，如果是对象，那么对象的数据即使发生变化，我们也无法监听，那么如果需要监听对象中的数据变化，就必须使用deep属性，并设置true

深度监听不推荐使用，因为它监听整个对象，比较消耗性能，所以我们可以指定去监听对象中的指定值。

但是需要注意，如果指定了对象中的某一个数据进行监听，那么我们写成字符串格式，deep属性可以省略

watch:{
	要监听的数据名:{
		handler(新值,旧值){},
		deep:true
	}
}

# 监听和计算属性的区别

  1. 监听函数无需调用，而计算属性需要调用，并且调用时不加()
  2. 监听函数监听的是属性的变化，拿变化后的属性去做后续的操作
  3. 计算属性是通过它依赖的属性发生变化后得到的值，并且数据的结果会被缓存，且函数中必须通过return返回 最终结果

当一个结果受多个属性影响的时候就需要使用计算属性 computed

当一个数据的变化参与到后续操作的时候需要使用监听 watch

# 路由模式

hash
	
	采用的是window.onhashchange=()=>{}实现的。在地址栏会出现#号，虽然看着不美观，但是不会包含在http请求中，对后端也没有任何影响

history
	history模式，利用了 HTML5 新增的 pushState() 和 replaceState() 方法，在原有的back、forward、go 的基础上，添加对历史记录修改的功能。history模式下有一个问题，就是当页面刷新时，如果后端没有做处理的话，就会因找不到资源而报404错误。

push方法会向history栈添加一条新纪录，成功跳转后点击浏览器的回退按钮会跳回之前的页面

replace方法不会向history栈添加纪录，成功跳转后点击浏览器的回退按钮会跳回上上个页面

back方法是后退，参数为0时是刷新，参数为1时是前进

# 生命周期

钩子函数：不需要手动调用，在某种条件满足时，会自动触发的函数就叫做钩子函数

# 初始

（创建期）把data和methods初始化，挂载到对象上

beforeCreate ：实例刚刚在内存中被创建出来，此时还没有初始化 data 和 methods 属性

created ：实例已经在内存中创建好，此时 data 和 methods 已经创建好，此时还没有开始编译模版

data methods 模版
beforeCreate undefined undefined undefined
created finish finish undefined

# 挂载

(挂载期)模版准备好之后，把数据填到模版上

beforeMount ：此时已经完成模版编译，但是还没有挂载到页面中

mounted ：这个时候已经把编译好的模版挂载到页面指定的容器里

beforeMount：data的数据可以访问和修改，而且此时的模版已经编译好了，还没有更新到页面中

mounted：此时编译的模版更新到页面中了

data methods 模版
beforeMount finish finish undefined
mounted finish finish finish

# 更新

beforeUpdate ：状态更新之前执行此函数，此时的data中的数据是最新，但是页面上显示的还是旧的，因为此时还没有开始重新渲染DOM节点

updated ：实例更新完毕之后调用此函数，此时data中的状态值和界面上显示的数据，都已经完成了更新，界面已经被重新渲染好了

beforeUpdate：此时修改输入框的内容，data中的数据已经更新了，但是页面上显示的还是旧数据

updated：此时data的内容已经更新，页面显示的也是最新的数据

# 销毁

beforeDestroy ：实例销毁之前调用，在这一步，实例仍然完全可用

destroyed ：实例销毁后调用，调用后，Vue实例指向的所有东西会被解绑，所有的事件监听器会被移除，所有的子实例也会被销毁

一般在 Vue2 里面双向绑定的原理用的是 Object.defineProperty ，但在 Vue3 里面升级为 new Proxy ，那么这两个有什么区别呢？就是在 Vue2 里面 Object.defineProperty 它是不支持监听引用类型的，也就是数组、对象这种应用类型是不能做响应式的，但是在 Vue3 里面 new Proxy 是可以监听引用类型的

  age: 10,

  () => obj.age,

  console.log("监听多个-new", newData);
  console.log("监听多个-new", oldData);
});

const gai = () => {
  num.value++;
  obj.age++;
};
</script>

// 动态生成路由
const getRouter = ()=>{
  // 其他的路由应该都是在登录成功的那一刻，动态生成的
  let children = new Array;
  // 拿到用户的权限列表
  let menu = store.getters.getNewMenu
  menu.map((item,index)=>{
    item.children.map((v,k)=>{
      // 小路由
      let itemrouter = {
        path:`/${item.name}/${v.name}`,
        name:v.name,
        component:()=>import(`../views/${item.name}/${v.name}.vue`)
      }
      children.push(itemrouter)
    })
  })
  let myRoute = {
    path:'/home',
    name:'home',
    component:()=>import(/* webpackChunName:"home"*/ '../views/home/home.vue'),
    children:children
  }
  // 动态添加路由
  router.addRoute(myRoute)
}
